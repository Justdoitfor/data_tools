# 招投标信息自动化采集与分析工具方案设计

## 一、 整体架构设计

工具整体分为四个逻辑层，确保高内聚低耦合：

### 1. 采集层 (Data Collection Layer)
- **职责**：从各个招投标平台（如中国政府采购网、省公共资源交易中心等）提取HTML或JSON数据。
- **核心组件**：
  - `RequestManager`：统一的网络请求发送模块，负责会话管理、代理轮换和请求重试。
  - `SiteSpiders`：针对不同站点的解析器，使用 `BeautifulSoup` 或 `lxml` 提取页面元素。
- **反爬策略说明**：
  - **请求头伪装**：随机使用不同操作系统的 `User-Agent`，携带必要的 `Referer`。
  - **IP代理池**：接入动态代理池，防止单IP高频访问被封禁。
  - **请求限速与抖动**：请求间隔加入随机延迟 (如 `sleep(1 ~ 3)` 秒)。
  - **动态渲染应对**：对需要执行JS的站点（如某些省份的交易中心）采用 `Playwright` 无头浏览器抓取。
  - **Cookie 管理**：定期更新或通过无头浏览器获取有效的 `Session/Cookie`。

### 2. 处理层 (Data Processing Layer)
- **职责**：将采集到的脏数据进行清洗、结构化、去重。
- **核心逻辑**：
  - **数据清洗**：剔除 HTML 标签残留，使用正则提取“预算金额”、“中标金额”（例如将“100.5万元”转为 `1005000` 的数值型数据）。
  - **结构化映射**：统一映射为标准字段（`项目名称`, `招标单位`, `中标单位`, `金额`, `发布时间`, `地区`, `项目类型`, `公告类型`, `URL`）。
  - **数据去重**：采用 `(项目名称 + 地区 + 公告类型)` 生成哈希值作为主键，或者基于余弦相似度进行标题去重（应对同一项目在多平台发布的现象）。

### 3. 分析层 (Data Analysis Layer)
- **职责**：对结构化的数据进行统计和关联计算。
- **核心功能**：
  - **多维排序**：支持按发布时间倒序、金额倒序等。
  - **主体关联分析**：统计同一中标单位在指定时间段内的中标频次和累计金额。
  - **区间与趋势分析**：按月/季度聚合数据，生成金额区间分布。
  - **商机推荐**：识别“高频发布招标的单位”和“招标金额热点地区”。

### 4. 展示层 (Presentation Layer)
- **职责**：提供用户交互入口和可视化报表。
- **实现方式**：
  - **CLI 命令行**：基于 `argparse` 或 `typer` 提供终端下的快速数据抓取和导出（CSV/Excel）。
  - **Web 演示页面**：基于 `Streamlit` 构建交互式看板。用户可输入关键字、时间、地区，实时展示表格、图表（Plotly / Altair 等），直观查看统计趋势。

## 二、 数据字典设计

| 字段名称     | 英文名称        | 数据类型 | 说明                           |
| ------------ | --------------- | -------- | ------------------------------ |
| 项目名称     | project_name    | String   | 招投标项目的完整名称           |
| 招标单位     | tenderer        | String   | 甲方/采购人                    |
| 中标单位     | winner          | String   | 乙方/中标方（招标公告中为空）  |
| 金额(元)     | amount          | Float    | 预算金额或中标金额             |
| 发布时间     | publish_date    | Date     | 公告发布时间 (YYYY-MM-DD)      |
| 地区         | region          | String   | 省市信息                       |
| 项目类型     | project_type    | String   | 货物/工程/服务                 |
| 公告类型     | notice_type     | String   | 招标公告/中标结果/变更公告     |
| 来源网址     | source_url      | String   | 原始公告链接                   |

## 三、 数据分析与可视化方案

1. **趋势图 (Line Chart)**
   - **X轴**：月份/日期
   - **Y轴**：招标/中标总金额（或项目数量）
   - **作用**：展示特定领域/关键词下的招投标活跃度时间趋势。

2. **排行榜 (Bar Chart)**
   - **维度**：中标单位频次 Top 10，或 招标单位发布金额 Top 10。
   - **作用**：挖掘高频招标方（潜在客户大户）和实力中标方（竞品分析）。

3. **金额区间分布 (Pie/Donut Chart)**
   - **区间**：<50万，50-200万，200-1000万，>1000万。
   - **作用**：了解该类型项目的普遍预算体量。

4. **地区分布图 (Map/Bar)**
   - **作用**：展示商机在全国/全省的空间分布热度。
